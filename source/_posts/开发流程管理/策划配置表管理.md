# 客户端配置数据管理

>开发或上线后不可避免的出现配置逻辑错误问题，系统功能通常会给策划留有很大的配置空间用于铺量和修改，但不可避免的会有错误，这里记录下配置表检查的原则

### 必须检查的情况
1. id 与其它表格关联
2. 配置资源路径，必须判断资源是否存在
3. 配置要求一定得格式，比如约定的分隔符 1;2 分割数组，可能会配置成1,2
4. 有 type:id  type 不同id 指向不用配置表，必须检查。
